# 9. IA (Classificação de tipo do documento)

Uso de **heurísticas** e **OpenAI** para classificar o tipo de cada documento no pdfsearchable.

---

## Objetivo

Atribuir automaticamente um **tipo** (doc_type) a cada documento indexado — por exemplo: contrato, nota_fiscal, relatório, procuração, petição, recibo, ata, certidão, e-mail, fatura, documento — para filtros no report e estatísticas por tipo.

---

## Modos (PDFSEARCHABLE_AI)

| Modo | Comportamento |
|------|----------------|
| **auto** (padrão) | Usa OpenAI se `OPENAI_API_KEY` estiver definida; caso contrário, usa heurísticas. |
| **heuristics** | Sempre heurísticas (rápido, sem custo, sem chamada externa). |
| **openai** | Sempre OpenAI (requer chave); em falha da API, fallback para heurísticas. |
| **ollama** | Modelo local Ollama (http://localhost:11434); em falha ou indisponível, fallback para heurísticas. Não requer dependência extra. |

Variável de ambiente: `PDFSEARCHABLE_AI`. Provedor alternativo: `PDFSEARCHABLE_AI_PROVIDER` (openai \| ollama).

---

## Heurísticas

- **Fonte:** `ai_classifier._classify_by_heuristics(text, path=None)`.
- **Lógica:** palavras-chave por tipo (ex.: “contrato”, “cláusula”, “partes” para contrato; “nota fiscal”, “nf-e”, “cfop” para nota_fiscal); pontuação com **peso por posição**: os primeiros ~1500 caracteres (título/cabeçalho) têm peso maior que o resto.
- **Nome do arquivo:** quando `path` é informado, o nome do arquivo pode reforçar o tipo (ex.: “contrato_xyz.pdf” → contrato).
- **Metadados do PDF (ID4):** quando disponíveis, título, assunto (subject) e keywords do PDF (fitz) são usados como hint no texto considerado para matching.
- **Fallbacks:** hints por regex (e-mail, fatura) e tipo genérico “documento” quando não há match.
- **Tipos suportados:** definidos em `DOC_TYPE_KEYWORDS` e em `KNOWN_TYPES` (contrato, nota_fiscal, relatório, procuração, petição, recibo, ata, certidão, comprovante, apresentação, manual, artigo, e-mail, fatura, documento).

---

## OpenAI (opcional)

- **Ativação:** definir `OPENAI_API_KEY` e (opcionalmente) `PDFSEARCHABLE_AI=openai` ou `auto`.
- **Dependência:** `pip install -e ".[ai]"` (openai).
- **Modelo:** `PDFSEARCHABLE_OPENAI_MODEL` (padrão: `gpt-4o-mini`).
- **Uso:** trecho do texto (e opcionalmente nome do arquivo) é enviado à API; a resposta é normalizada para um dos `KNOWN_TYPES`; em caso de erro ou resposta inválida, fallback para heurísticas.
- **Resultado:** `ClassificationResult(label, source="openai")`; no índice e no report, documentos classificados pela IA aparecem com o indicador **(IA)**.

---

## Ollama (modelo local)

- **Ativação:** `PDFSEARCHABLE_AI=ollama`. Servidor Ollama em execução (ex.: `ollama serve`).
- **URL:** `PDFSEARCHABLE_OLLAMA_URL` (padrão: `http://localhost:11434`).
- **Modelo:** `PDFSEARCHABLE_OLLAMA_MODEL` (padrão: `llama3.2`).
- **Uso:** mesmo prompt de classificação que a OpenAI; resposta normalizada para `KNOWN_TYPES`; em erro ou timeout, fallback para heurísticas.
- **Resumo (IA9):** em modo `ollama`, o indexador gera também um resumo de 1–3 frases via Ollama e armazena em `summary` no índice (exibido no report e no document-view).
- **Metadados extraídos (Ollama):** com `PDFSEARCHABLE_AI=ollama`, o indexador chama `extract_metadata_ollama(text)` para obter, numa única chamada, o máximo de informações estruturadas em **20 categorias**: CPFs, CNPJs, e-mails, IPs, **URLs**, **domínios**, **telefones**, **endereços completos**, **CEPs**, **processos judiciais**, **placas de veículos**, **RGs**, **protocolos**, **hashes criptográficos**, **coordenadas GPS**, **timestamps**, **referências legais (leis/decretos)**, valores monetários, partes e datas. Os resultados são **mesclados** com a extração por regex (16 tipos de entidades com deduplicação por lowercase); endereços e telefones vêm apenas do Ollama e são gravados em `identified_addresses` e `identified_phones` no índice, exibidos na sidebar do document-view e no report.
- **Resultado:** `ClassificationResult(label, source="ollama")`; no report e no status, exibido **(IA)** como na OpenAI.
- **Busca (CLI):** com `--ollama`, o comando `pdfsearchable search "termo"` expande a consulta via Ollama (sinônimos/termos relacionados) e executa a busca FTS com o termo original **OU** os termos expandidos. Padrão: desativado. Ver [03-CLI.md — search](03-CLI.md#search--pesquisar).
- **RAG (perguntas):** o comando `pdfsearchable ask <id ou nome> "pergunta"` envia o texto do documento e a pergunta ao Ollama e exibe a resposta. O servidor `pdfsearchable serve` expõe o endpoint `/api/ask` (JSON com `id` e `question`) para a visualização de documento no report. Ver [03-CLI.md — ask](03-CLI.md#ask--perguntar-sobre-um-documento-rag).

---

## Interface pública

- **Função:** `classify_document(text, path=None, metadata_hint=None)`.
- **Retorno:** `ClassificationResult` com:
  - **label:** string do tipo (ex.: "contrato", "nota_fiscal").
  - **source:** `"heuristics"`, `"openai"` ou `"ollama"`.
- **metadata_hint:** dict opcional com `title`, `subject`, `keywords` do PDF para reforçar a classificação (ID4).
- **confidence:** valor entre 0 e 1. Para heurísticas é calculado pela pontuação relativa entre tipos candidatos. Para Ollama é fixado em `0.85` (indicador de "alta confiança LLM"). Gravado no índice como `classification_confidence` e exibido no report (percentagem ao lado do tipo).

O indexador chama `classify_document(full_text, pdf_path, metadata_hint=...)` quando o usuário não informa tipo explícito; o `label` é gravado em `doc_type`, o `source` em `classification_source` e o `confidence` em `classification_confidence`. Em modo ollama, são ainda extraídos resumo (IA9), tags (ID5), valores monetários (ID8) e partes/participantes (ID9) e armazenados no índice. Com `PDFSEARCHABLE_AI=heuristics` nenhuma chamada a API externa é feita (zero dependência externa).

## Variáveis de ambiente

| Variável | Descrição |
|----------|-----------|
| `PDFSEARCHABLE_AI` | `auto`, `heuristics`, `openai` ou `ollama`. |
| `PDFSEARCHABLE_AI_PROVIDER` | `openai` ou `ollama` (provedor alternativo). |
| `OPENAI_API_KEY` | Chave da API OpenAI (ou compatível). |
| `PDFSEARCHABLE_OPENAI_MODEL` | Modelo OpenAI (ex.: `gpt-4o-mini`). |
| `PDFSEARCHABLE_OLLAMA_URL` | URL da API Ollama (padrão: `http://localhost:11434`). |
| `PDFSEARCHABLE_OLLAMA_MODEL` | Modelo Ollama para RAG/ask/resumo (ex.: `llama3.2`). |
| `PDFSEARCHABLE_OLLAMA_CLASSIFY_MODEL` | Modelo Ollama alternativo **só para classificação** (ex.: `llama3.2:1b`). Mantém `OLLAMA_MODEL` para tarefas longas. |
| `PDFSEARCHABLE_OLLAMA_KEEP_ALIVE` | Tempo que o Ollama mantém o modelo em memória entre chamadas (`5m`, `30m`, `1h`, `-1`=permanente, `0`=descarregar). Padrão: `5m`. |
| `PDFSEARCHABLE_CLASSIFIER_FEEDBACK` | `1` = grava amostras rotuladas (label + trecho) em `.pdfsearchable/feedback.jsonl` para fine-tuning futuro. |
| `PDFSEARCHABLE_DETECT_REDACTIONS` | `1` = detecta zonas redactadas (tarjas pretas). Ver [10-config.md](10-config.md#detecções-opcionais-redactions-forensics-contratos). |
| `PDFSEARCHABLE_FORENSICS` | `1` = análise forense do PDF (revisões incrementais, produtor, anomalias). |
| `PDFSEARCHABLE_CONTRACTS` | `1` = detecção heurística de cláusulas contratuais. |

---

## Exibição no projeto

- **CLI status:** coluna “Tipo” com doc_type; quando source é openai ou ollama, exibido “(IA)” ao lado.
- **Report:** badge do tipo por documento; “(IA)” quando classificação veio da OpenAI ou Ollama; resumo e tags quando disponíveis; percentual de texto OCR (ID11); filtro “Por tipo” e nuvem por tipo usam `doc_type`.
- **Document-view:** resumo, tags, valores monetários, partes/participantes e percentual OCR quando disponíveis.
